## 修复 iPad OCR 图片识别失败

- 根因：iPad/iOS 默认照片格式为 HEIC/HEIF，PaddleOCR 底层 PIL/cv2 无法解码该格式 → OCR 返回空文本 → 后端返回 422 "未能从图片中识别到文字"。
- 修复方案（前端 Canvas 转换层）：
  1. 新增 `convertImageToJPEG(file, maxWidth=2000)` 函数：通过 FileReader + Image + Canvas 将任意格式图片统一转为 JPEG（含白底填充、大图压缩），转换失败时 fallback 用原文件。
  2. `ocrImageFile` 函数在上传前先调 `convertImageToJPEG`，确保 PaddleOCR 收到的始终是 JPEG。
- 后端容错（`api/questions.py` OCR 端点）：
  1. `content_type` 为空时放行（部分移动端浏览器上传时 type 为空），仅当有值且不以 `image/` 开头时才拒绝。
  2. 增加 `except Exception` 捕获 OCR 处理异常，打印完整 traceback 并返回 500 含错误信息，方便排查。
- commit `f591064`，已 push GitHub。
- 服务器生效：`cd ~/Interview_agent && git pull && sudo systemctl restart gt-agent`。

## OCR 全平台失败排查（电脑端也开始 422）

- 用户反馈：不仅 iPad，电脑端截图和图片文件也全部 422。
- 根因发现：`core/ocr.py` 的 `extract_text_from_image` 用 `except Exception` 吞掉所有异常并返回空字符串，导致 PaddleOCR 的任何错误（内存不足、模型加载失败、进程崩溃等）都被隐藏，用户只看到 422"未识别到文字"。
- 修复（commit `d6266a3`）：移除 ocr.py 的 try-except，让异常冒泡到 API 层（`api/questions.py` 的 `except Exception` 会打印 traceback 并返回 500 含具体错误信息）。
- 排查方向：PaddleOCR 很可能因服务器内存不足或进程崩溃导致初始化/推理失败，需查看 `journalctl -u gt-agent` 日志确认。

## numpy 版本冲突排查（重要教训）

- paddlepaddle 2.6.2 **不兼容 numpy 2.x**：内部使用 `np.sctypes`（numpy 2.0 已删除），运行时报 `np.sctypes was removed in the NumPy 2.0 release`。
- opencv-python-headless 5.x 要求 numpy>=2，4.6.x 兼容 numpy 1.x。
- scikit-learn 1.3.2 要求 numpy<2.0。
- 最终锁定版本组合（commit `649604c`）：numpy==1.26.4 + scikit-learn==1.3.2 + paddlepaddle==2.6.2 + paddleocr==2.7.3 + opencv-python-headless==4.6.0.66。
- 教训：paddlepaddle 2.x 系列只能配 numpy 1.x，不要尝试升 numpy 2.x。如果需要 numpy 2.x，必须升级到 paddlepaddle 3.x（但 3.x 更重，轻量服务器不推荐）。

## 最终方案：弃用 paddlepaddle，换 RapidOCR (ONNX)

- paddlepaddle 2.6.2 预编译包使用 AVX 指令，腾讯云轻量服务器 CPU 不支持 → `SIGILL (Illegal instruction)` 崩溃，服务直接 core dump。
- 换用 `rapidocr-onnxruntime`（commit `67a8c09`）：
  - 使用 PaddleOCR 同款模型，走 ONNX Runtime CPU 推理
  - 不依赖 paddlepaddle，无需 AVX 指令集
  - 内存占用约 300MB，适合轻量服务器
  - API：`RapidOCR()` → `result, elapse = ocr(img_path)` → `result = [[box, text, score], ...]`
- requirements.txt 去掉 paddlepaddle/paddleocr，加 rapidocr-onnxruntime。
- 服务器需卸载 paddlepaddle/paddleocr（可选）+ 安装 rapidocr-onnxruntime。
