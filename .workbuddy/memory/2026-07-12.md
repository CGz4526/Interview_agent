## 安全架构发现（auth.py 代码层）
- `get_current_user`（api/auth.py 第 103–135 行）设计上**无 token 时兜底返回预设用户**（`PRESET_USER` or "111"）。即系统本质是"无登录模式"：即使设了强密码，直接调 API 不带 token 仍放行。
- `ALLOW_REGISTER`（第 28、140 行）确实生效：false 时 /register 返回 403。
- 仅改 .env（PUBLIC_MODE=true+强密码+关注册）只能挡"用前端登录"的场景；要彻底防陌生人的纯 API 访问，需在 PUBLIC_MODE=true 时让 `get_current_user` 无 token 返回 401，或叠加 Nginx+HTTPS。

## 安全加固修复（本次完成）
- 根因：`frontend/index.html` 的 `init()` 直接 `showApp()`（原注释"取消登录模式"），导致即使后端有鉴权，网站也跳过登录直接进控制台；并非仅"无 token 兜底"。
- 改动（commit `6644db7`，已 push）：
  1. `api/auth.py`：新增 `PUBLIC_MODE` 读取；`get_current_user` 在 `PUBLIC_MODE=true` 且无有效 token 时返回 401；本地模式(`false`)仍兜底免登录，向后兼容。
  2. `frontend/index.html`：`init()` 改为先 `api('/api/auth/me')` 校验登录态，成功才 `showApp()`，失败（公开模式 401）由 `api()` 自动 `showLogin()`。
- 效果：本地模式免登录不变；公开模式强制登录，纯 API 无 token 访问一律 401。
- 服务器生效步骤：`cd ~/Interview_agent && git pull && sudo systemctl restart gt-agent`（注意先保存 nano 里的 .env 改动：Ctrl+O / Enter / Ctrl+X）。
